## A01: 工具链重建 (2026-04-23)

**问题**: VS2022 NMake 不存在，原 MSYS2 安装被中断，gcc/cmake 缺 DLL。

**解决方案**:
- MSYS2 完整安装通过 pacman 安装了 mingw64 工具链
- 通过 bash 调用 MinGW 程序（bash 自动设置正确的 DLL 搜索路径）
- pybind11 通过 pip 安装到 Python314 site-packages
- CMake 指定 pybind11_DIR 参数

**构建命令**:
```bash
export PATH="/mingw64/bin:/usr/bin:$PATH"
cd /c/gitcloud/auto-lorr-new/lorr-code
rm -rf build
cmake -B build -G "MinGW Makefiles" . \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_C_COMPILER=gcc \
    -DCMAKE_CXX_COMPILER=g++ \
    -DBoost_USE_STATIC_LIBS=OFF \
    -DPYTHON_EXECUTABLE=C:/Python314/python.exe \
    -DPYTHON_LIBRARY=C:/Python314/libs/python314.lib \
    -DPYTHON_INCLUDE_DIR=C:/Python314/include \
    -Dpybind11_DIR="C:/Users/take_/AppData/Roaming/Python/Python314/site-packages/pybind11/share/cmake/pybind11"
cmake --build build --target lifelong
```

**结果**: ✅ BUILD SUCCESS - lifelong.exe 3,698,024 bytes
